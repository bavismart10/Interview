Laravel Interview Questions

Laravel Basics

1. What is Laravel?
Laravel is an open-source PHP framework based on MVC architecture designed for fast, secure, and maintainable web development.
2. What is MVC?
Model → Handles database & business logic.
View → Handles UI (Blade).
Controller → Handles request & response.
3. What is Artisan?
Artisan is Laravel's command line tool. 
php artisan make:controller UserController
php artisan make:model User -m
4. What is Composer?
Composer is a dependency manager for PHP used to install Laravel packages.

Routing & Controllers

5. What is Routing?
Routing maps URLs to controllers or closures. 
Route::get('/users', [UserController::class, 'index']);
6. What is Middleware?
Middleware filters HTTP requests before reaching controllers (Auth, Role check). 
Route::get('/dashboard', function () {
    return view('dashboard');
})->middleware('auth');

Eloquent ORM

7. What is Eloquent?
Eloquent is Laravel's Active ORM (Object-Relational Mapping) used to interact with database using PHP models instead of writing raw SQL queries. 
$users = User::where('status',1)->get();
8. find() vs first() vs get()
find() → By primary key.
first() → First matching record.
get() → Multiple records. 
User::find(1);
User::where('status',1)->first();
User::where('status',1)->get();
9. N+1 Problem?
Multiple queries inside a loop due to lazy loading.
❌ Problem: 
$users = User::all();
foreach($users as $user){
    echo $user->posts;
}
✅ Solution (Eager Loading): 
$users = User::with('posts')->get();

Authentication & Security

10. How Laravel handles Authentication?
Using Breeze, Jetstream, Sanctum. 
if(Auth::attempt(['email'=>$email,'password'=>$password])){
    return redirect()->intended('dashboard');
}
11. What is CSRF (Cross-Site Request Forgery), and how does Laravel prevent it?
Using csrf tokens to Prevents unauthorized form submissions. 

    @csrf

Queue & Jobs

12. What is Queue?
Queues process background tasks like emails. 
php artisan make:job SendEmailJob

class SendEmailJob implements ShouldQueue
{
    public function handle(){
        \Log::info('Email sent');
    }
}
Dispatch Job: 
SendEmailJob::dispatch();

Cache & Optimization

13. What is Caching?
Stores frequently used data to improve performance. It stores query results without hitting the database every time. 
Cache::remember('users',60,function(){
    return User::all();
});
14. How to optimize Laravel?
Use caching, eager loading, indexing, queues, config cache. 
php artisan config:cache
php artisan route:cache

Advanced Concepts

15. What is Dependency Injection?

Laravel automatically injects class dependencies.

Dependency Injection is a technique where a class receives its dependencies from outside instead of creating them inside the class.

public function __construct(UserService $service){
    $this->service = $service;
}
16. What is Soft Delete?
Soft delete marks record using deleted_at column instead of deleting permanently. 
use SoftDeletes;
User::withTrashed()->get();
User::restore();
17. What is Database Transaction?
Ensures all queries succeed or rollback. 
DB::transaction(function(){
    Order::create([...]);
    Payment::create([...]);
});
18. How do you measure performance in Laravel?
Benchmark Helper.
19. What is Benchmark in Laravel?
Benchmark in Laravel is a built-in helper used to measure the execution time of a piece of code and dump the result with timing information.
I use Laravel's Benchmark helper to measure and compare execution time before optimizing. 
use Illuminate\Support\Benchmark;

Benchmark::dd(function () {
    return User::all();
});
20. Explain the lifecycle of a Laravel request?

1. Entry Point → All requests enter through public/index.php.

2. Bootstrap → Laravel loads configuration and bootstraps the application.

3. Service Providers → All registered service providers are loaded.

4. Middleware → Request passes through global and route middleware.

5. Routing → Router matches the request to a route.

6. Controller / Closure → The matched controller method or closure executes.

7. Response → A response is generated (view, JSON, redirect, etc.).

8. Send Response → Response is sent back to the browser.

21. What's the difference between a Service Container and a Service Provider in Laravel?

Service Container → The dependency injection container that manages class dependencies and resolves objects automatically.

app()->make(UserService::class);

Service Provider → A place to register bindings, services, and application bootstrapping logic into the service container.

public function register()
{
    $this->app->bind(UserService::class, function ($app) {
        return new UserService();
    });
}

Simple Difference:

• Service Container → Stores and resolves dependencies.

• Service Provider → Registers services into the container.
