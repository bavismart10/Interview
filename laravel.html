Laravel Interview Questions

Laravel Basics

1. What is Laravel?
Laravel is an open-source PHP framework based on MVC architecture designed for fast, secure, and maintainable web development.
2. What is MVC?
Model → Handles database & business logic.
View → Handles UI (Blade).
Controller → Handles request & response.
3. What is Artisan?
Artisan is Laravel's built-in command-line interface (CLI). It is used to create controllers, models, migrations, and run tasks. 
# Create a model
php artisan make:model User

# Create a model with migration
php artisan make:model User -m 

# Run migrations
php artisan migrate

#Model + Controller + Resource
php artisan make:mcr Post
4. What is Composer?
Composer is a dependency manager for PHP used to install Laravel packages.

Routing & Controllers

5. What is Routing?
Routing maps URLs to controllers or closures. 
Route::get('/users', [UserController::class, 'index']);
6. What is Middleware?
Middleware filters HTTP requests before reaching controllers (Auth, Role check). 
Route::get('/dashboard', function () {
    return view('dashboard');
})->middleware('auth');

Eloquent ORM

7. What is Eloquent?
Eloquent is Laravel's Active ORM (Object-Relational Mapping) used to interact with database using PHP models instead of writing raw SQL queries. 
$users = User::where('status',1)->get();
8. find() vs first() vs get()
find() → Used to fetch a record by primary key.
first() → Used to fetch the first record from the query based on a condition.
get() → Used to fetch all records matching the query. 
User::find(1);  (OR)  SELECT * FROM users WHERE id = 1 LIMIT 1;
User::where('status',1)->first();  (OR)  SELECT * FROM users WHERE email = 'test@example.com' LIMIT 1;
User::where('status',1)->get();   (OR)   SELECT * FROM users WHERE status = 'active';
9. N+1 Problem?
Multiple queries inside a loop due to lazy loading.
❌ Problem: 
$users = User::all();
foreach($users as $user){
    echo $user->posts;
}
✅ Solution (Eager Loading): 
$users = User::with('posts')->get();

Queue & Jobs

12. What is Queue, Job, Dispatch, Queue Worker, and Handle?
Job: A class that defines a task to be executed by the queue. Example: Sending an email to a user. 
// Create a Job
php artisan make:job SendEmailJob

File location: app/Jobs/SendEmailJob.php
Dispatch: Sends the job to the queue for background processing. Example: Dispatch the email job after user registration. 
SendEmailJob::dispatch();
Queue: Queue is a system where jobs (tasks) are stored and run in the background asynchronously.
Example: Sending emails or generating reports without making the user wait. 
Queue stores tasks until a worker processes them in the background.

Config: config/queue.php
Database (for database driver): database/migrations/xxxx_xx_xx_create_jobs_table.php
Queue Worker: A background process that continuously checks the queue for new jobs and executes them. Example: Worker picks jobs from the queue and runs them. 
php artisan queue:work
Handle: The method inside the job class that contains the actual logic to perform the task. Example: Sending the email, generating report, or logging info. 
class SendEmailJob implements ShouldQueue
{
    public function handle() {
        \Log::info('Email sent');
    }
}
Process Flow: 
1. Create a Job -> 2. Dispatch the Job -> 3. Store in Queue -> 
4. Queue Worker Picks Job -> 5. Job Execution (handle()) -> 6. Job Completion

Cache & Optimization

13. What is Caching?
Stores frequently used data to improve performance. It stores query results without hitting the database every time. 
Cache::remember('users',60,function(){
    return User::all();
});
14. How to optimize Laravel?
Use caching, eager loading, indexing, queues, config cache. 
php artisan config:cache
php artisan route:cache

Advanced Concepts

1. What is Dependency Injection and Dependency Injection Container in Laravel?
Dependency Injection (DI): In Laravel, a class receives dependencies from outside instead of creating them inside. Example: 
class UserController extends Controller
{
    protected $mailer;

    public function __construct(Mailer $mailer)
    {
        $this->mailer = $mailer; // dependency injected
    }

    public function notify()
    {
        $this->mailer->send("Welcome User!");
    }
}
Dependency Injection Container (Service Container): In Laravel, a system (the container) automatically creates and manages dependencies and injects them wherever needed. Example: 
namespace App\Http\Controllers;
use App\Services\UserService;

// Laravel automatically resolves UserService and injects it
public function __construct(UserService $service){
    $this->service = $service;
}
public function index() {
    $this->service->doSomething();
}
2. What is Soft Delete?
Soft delete marks record using deleted_at column instead of deleting permanently. 
use SoftDeletes;
User::withTrashed()->get();
User::restore();
3. What is Database Transaction?
A database transaction is a sequence of one or more database operations that are treated as a single unit. Ensures all queries succeed or rollback. 
use Illuminate\Support\Facades\DB;

DB::beginTransaction();

try {
    DB::table('accounts')->where('id', 1)->decrement('balance', 100);
    DB::table('accounts')->where('id', 2)->increment('balance', 100);

    // If all queries succeed
    DB::commit();
} catch (\Exception $e) {
    // If any query fails
    DB::rollBack();
    \Log::error('Transaction failed: ' . $e->getMessage());
}
4. How do you measure performance in Laravel?

Laravel Debugbar : - Shows execution time, SQL queries, memory usage, route info, and more in the browser.
Purpose : Queries with execution time, memory usage, and performance issues during development. 

// Install Debugbar (only in dev)
composer require barryvdh/laravel-debugbar --dev

use Debugbar;

\Debugbar::info('This is debug info');
\Debugbar::startMeasure('loading', 'Loading users');

$users = User::all();

\Debugbar::stopMeasure('loading');
\Debugbar::warning('Loaded ' . count($users) . ' users');

Benchmark Helper : - Benchmark in Laravel is a built-in helper used to measure the execution time of a piece of code and dump the result with timing information.
Purpose : Analyze performance of individual operations to optimize code. 

// Using microtime
$start = microtime(true);

// Code to benchmark
$users = User::all();

$end = microtime(true);
\Log::info('Execution time: ' . ($end - $start) . ' seconds');

DB::listen : - Logs all executed SQL queries with their execution time.
Purpose : Identify slow queries and optimize database performance. 

use Illuminate\Support\Facades\DB;
use Illuminate\Support\Facades\Log;

DB::listen(function ($query) {
    Log::info(
        $query->sql . ' [' . implode(',', $query->bindings) . '] - ' . $query->time . 'ms'
    );
});

6. Explain the lifecycle of a Laravel request?

1. Entry Point → All requests enter through public/index.php.

2. Bootstrap → Laravel loads configuration and bootstraps the application.

3. Service Providers → All registered service providers are loaded.

4. Middleware → Request passes through global and route middleware.

5. Routing → Router matches the request to a route.

6. Controller / Closure → The matched controller method or closure executes.

7. Response → A response is generated (view, JSON, redirect, etc.).

8. Send Response → Response is sent back to the browser.

7. What's the difference between a Service Container and a Service Provider in Laravel?

Service Container → The dependency injection container that manages class dependencies and resolves objects automatically.

app()->make(UserService::class);

Service Provider → A place to register bindings, services, and application bootstrapping logic into the service container.

public function register()
{
    $this->app->bind(UserService::class, function ($app) {
        return new UserService();
    });
}

Simple Difference:

• Service Container → Stores and resolves dependencies.

• Service Provider → Registers services into the container.
