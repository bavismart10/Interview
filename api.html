PHP API Interview Questions

Basic Concepts

1. What is a Web API?
A Web API allows communication between client and server over HTTP using JSON or XML.
2. What is a RESTful API?
A RESTful API follows REST principles and uses HTTP methods like GET, POST, PUT, DELETE.
3. Difference between REST and SOAP?
REST → Lightweight, mostly JSON.
SOAP → Heavy protocol, uses XML and strict standards.
4. What are HTTP Methods?
GET (Read), POST (Create), PUT (Update), DELETE (Remove). 
GET /users
POST /users
PUT /users/1
DELETE /users/1
5. What are HTTP Status Codes?
200 OK
201 Created
400 Bad Request
401 Unauthorized
403 Forbidden
404 Not Found
500 Internal Server Error
6. What is JSON?
JSON is a lightweight data format used for API communication. 
{
  "name": "Bavithran",
  "role": "PHP Developer"
}
7. What is Authentication in API?
Authentication verifies user identity using tokens like JWT or API keys.
8. What is Statelessness in REST?
Each request contains all necessary information. Server does not store client state.

PHP API Implementation

9. How to return JSON response in PHP?
Use header() and json_encode(). 
header("Content-Type: application/json");
$data = ["status" => "success"];
echo json_encode($data);
10. How to receive JSON request in PHP?
Use php://input and json_decode(). 
$data = json_decode(file_get_contents("php://input"), true);
echo $data['name'];
11. What is CORS?
CORS allows or blocks requests from different domains. 
header("Access-Control-Allow-Origin: *");
header("Access-Control-Allow-Methods: GET, POST, PUT, DELETE");
12. What is CRUD in API?
CRUD → Create, Read, Update, Delete.

Security & Best Practices

13. How to prevent SQL Injection?
Use prepared statements and parameter binding. 
$stmt = $conn->prepare("SELECT * FROM users WHERE email = ?");
$stmt->execute([$email]);
14. What is JWT?
JWT (JSON Web Token) is used to securely transmit authentication data.
15. What is Rate Limiting?
Limits number of API requests per user/IP to prevent abuse.
16. What is API Versioning?
Maintaining multiple API versions like:
/api/v1/users
/api/v2/users
17. What is 401 vs 403?
401 → Not authenticated.
403 → Authenticated but no permission.

Advanced / Interview Tricky Questions

18. Difference between PUT and PATCH?
PUT → Updates full resource.
PATCH → Updates partial resource.
19. What is Idempotent Method?
Repeated requests produce the same result.
GET, PUT, DELETE are idempotent.
20. What is Pagination in API?
Pagination limits large responses into smaller pages. 
SELECT * FROM users LIMIT 10 OFFSET 0;
21. How to design secure and scalable API?
• Use HTTPS
• Use Token/JWT Authentication
• Validate input
• Use Prepared statements
• Implement Rate limiting
• Use Pagination & Caching
• Move heavy tasks to Queue
