OAuth2 & JWT Authentication Interview Questions

OAuth2 Authentication

1. What is OAuth2?
OAuth2 is an authorization framework that defines how tokens are issued, refreshed, and used.
2. Core Concepts in OAuth2
Resource Owner → User
Client → Web/Mobile App
Authorization Server → Issues Tokens
Resource Server → API that validates tokens
3. Common OAuth2 Flows
Authorization Code (Most secure & common)
Client Credentials (Machine-to-machine)
Password Grant (Legacy / Discouraged)
Refresh Token Flow
4. OAuth2 Example Flow
1. User logs in via Authorization Server
2. Authorization Server issues Access Token
3. Client sends token to API
4. API validates token and returns data

JWT (JSON Web Token)

5. What is JWT?
JWT is a secure token format used to carry authenticated user information in APIs.
6. JWT Structure
HEADER.PAYLOAD.SIGNATURE
7. JWT Components
Header → Algorithm & token type
Payload → Claims (id, role, exp, iss)
Signature → Ensures integrity and prevents tampering
8. Basic JWT Creation Example in PHP
$header = base64_encode(json_encode(["alg"=>"HS256","typ"=>"JWT"]));
$payload = base64_encode(json_encode([
    "id"=>1,
    "role"=>"admin",
    "exp"=>time()+3600
]));
$signature = hash_hmac("sha256", "$header.$payload", "secretKey", true);
$jwt = "$header.$payload.".base64_encode($signature);
echo $jwt;

OAuth2 vs JWT (Important Difference)

9. Difference Between OAuth2 and JWT
OAuth2 → Authorization framework
JWT → Token format

OAuth2 defines flows & roles
JWT encodes user data

OAuth2 can use JWT or opaque tokens
JWT is often used inside OAuth2
10. Important Notes
• OAuth2 can exist without JWT
• JWT can exist without OAuth2

JWT Validation Process

11. How JWT is Sent in API Request?
Authorization: Bearer <JWT>
12. How Server Validates JWT?
• Verify signature (secret/public key)
• Check expiration (exp)
• Validate claims (iss, aud, nbf)

If valid → Request proceeds
If invalid → 401 Unauthorized

Token Storage Best Practices

13. Where to Store Tokens?
Access Token → Short-lived, sent via Authorization header
Refresh Token → Stored securely (HttpOnly cookie / secure storage)
Never expose refresh tokens to JavaScript
14. What is Access Token vs Refresh Token?
Access Token → Used to access APIs (short lifespan)
Refresh Token → Used to generate new access tokens
