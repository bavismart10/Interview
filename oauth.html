OAuth2 & JWT Authentication Interview Questions

OAuth2 Authentication

1. What is OAuth 2.0?
OAuth 2.0 is an authorization framework that allows third-party applications to access user resources without sharing the user's credentials (username & password).
2. Why OAuth 2.0 is used?

To provide secure and controlled access to APIs without exposing user credentials.

Example: Login with Google, Facebook, GitHub.

3. Core Concepts / Key Roles in OAuth 2.0

1. Resource Owner -> User

2. Client -> Application requesting access (Web/Mobile App)

3. Authorization Server -> Authenticates user and provides token (Issues Tokens)

4. Resource Server -> API that validates tokens

4. OAuth 2.0 Process Flow (Simple Steps)?

1. User clicks "Login with Google".

2. User is redirected to Authorization Server.

3. User logs in and gives permission.

4. Authorization Server sends Authorization code to the client.

5. Client exchanges code for Access Token.

6. Client uses token to access user data.

Authorization Server = Google
Client = App / Web
5. What is Access Token?

A short-lived token used to access protected resources from APIs.

6. What is Refresh Token?

A long-lived token used to get a new access token when the current one expires.

JWT (JSON Web Token)

1. What is JWT?
JWT (JSON Web Token) is a secure token format used for authentication and exchange information between client and server..
2. Structure of JWT?

JWT consists of 3 parts:

1. Header -> Algorithm & token type

2. Payload -> User data & claims

3. Signature -> Security validation

Format: Header.Payload.Signature

3. JWT Components
Header → Algorithm & token type
Payload → Claims (id, role, exp, iss)
Signature → Ensures integrity and prevents tampering
4. Basic JWT Creation Example in PHP
$header = base64_encode(json_encode(["alg"=>"HS256","typ"=>"JWT"]));
$payload = base64_encode(json_encode([
    "id"=>1,
    "role"=>"admin",
    "exp"=>time()+3600
]));

// $signature ensures the token cannot be changed by anyone without knowing secretKey.
$signature = hash_hmac("sha256", "$header.$payload", "secretKey", true);
$jwt = "$header.$payload.".base64_encode($signature);
echo $jwt;
5. JWT Authentication Flow (Simple Steps)?

1. User logs in with username and password.

2. Server validates credentials.

3. Server generates JWT token.

4. Token is sent to client.

5. Client stores token (localStorage / cookies).

6. Client sends token with every API request.

7. Server verifies token and returns response.

6. Where JWT is stored?

LocalStorage, SessionStorage, or Secure HTTP-only Cookies.

7. Difference between Session and JWT?

Session -> Stored on server, stateful.

JWT -> Stored on client, stateless.

8. Advantages of JWT?

Stateless, scalable, faster authentication, microservice friendly.

OAuth2 vs JWT (Important Difference)

1. Difference between OAuth 2.0 and JWT?

OAuth 2.0 -> Authorization framework (process).

JWT -> Token format (data structure).

OAuth uses JWT as access tokens in many systems.

2. Important Notes
• OAuth2 can exist without JWT
• JWT can exist without OAuth2

JWT Validation Process

11. How JWT is Sent in API Request?
Authorization: Bearer <JWT>
12. How Server Validates JWT?
• Verify signature (secret/public key)
• Check expiration (exp)
• Validate claims (iss, aud, nbf)

If valid → Request proceeds
If invalid → 401 Unauthorized

Token Storage Best Practices

13. Where to Store Tokens?
Access Token → Short-lived, sent via Authorization header
Refresh Token → Stored securely (HttpOnly cookie / secure storage)
Never expose refresh tokens to JavaScript
14. What is Access Token vs Refresh Token?
Access Token → Used to access APIs (short lifespan)
Refresh Token → Used to generate new access tokens
