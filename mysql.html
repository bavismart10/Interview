SQL & MySQL Interview Questions

Basic Concepts

1. What is SQL?
SQL (Structured Query Language) is used to store, retrieve, update, and delete data from databases. 
SELECT * FROM users;
2. What is MySQL?
MySQL is an open-source relational database management system that uses SQL to manage data.
3. Difference between SQL and MySQL?
SQL is a language.
MySQL is a database system that uses SQL.
4. What is Normalization?
Normalization is a database design technique that organizes data into multiple related tables to reduce redundancy and improve accuracy.

✅ Minimize data redundancy

✅ Improve data integrity

✅ Make the database more efficient and maintainable

5. What is Denormalization?
Denormalization combines related tables to reduce joins and improve read performance.

✅ Make queries faster

✅ Reduce complex joins

✅ Improve read performance

6. Difference between OLTP and OLAP?
OLTP → Transactional, high insert/update operations.
OLAP → Analytical, complex queries, reporting.

Queries & Commands

1. What is a Primary Key?
A primary key uniquely identifies each record in a table and does not allow NULL values. 
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100)
);
2. What is a Foreign Key?
A foreign key creates a relationship between two tables. 
ALTER TABLE orders
ADD CONSTRAINT fk_user
FOREIGN KEY (user_id) REFERENCES users(id);
3. Difference between DELETE, TRUNCATE, DROP?

DELETE → Removes selected rows, can use WHERE.

TRUNCATE → Removes all rows, faster, resets auto_increment.

DROP → Deletes the entire table with data structure.

DELETE FROM users WHERE id = 1;
TRUNCATE TABLE users;
DROP TABLE users;
4. What is WHERE clause?
Filters records based on conditions. 
SELECT * FROM users WHERE status = 1;
5. Difference between WHERE and HAVING?
WHERE filters rows before grouping.
HAVING filters after GROUP BY. 
SELECT department, COUNT(*)
FROM employees
GROUP BY department
HAVING COUNT(*) > 5;
6. What is GROUP BY?
Groups rows with the same values and is used with aggregate functions like COUNT, SUM, AVG, MIN, MAX.
7. What are Aggregate Functions?
COUNT(), SUM(), AVG(), MIN(), MAX() 
SELECT COUNT(*), SUM(salary), AVG(salary)
FROM employees;
8. Difference between DISTINCT and GROUP BY?
DISTINCT removes duplicate rows.
GROUP BY groups rows for aggregation.

Joins

1. What is JOIN?
JOIN combines rows from two or more tables based on a related column. 
SELECT users.name, orders.amount
FROM users
INNER JOIN orders ON users.id = orders.user_id;
2. Types of JOINs?
INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL JOIN.

INNER JOIN -> Returns only rows with matching values in both tables.

LEFT JOIN -> Returns all rows from the left table and matching rows from the right table.

RIGHT JOIN -> Returns all rows from the right table and matching rows from the left table.

FULL JOIN -> Returns all rows from both tables, with NULLs where there is no match.

3. Difference between INNER JOIN and LEFT JOIN?
INNER JOIN → Returns matching records only.
LEFT JOIN → Returns all records from left table. 
SELECT *
FROM users
LEFT JOIN orders ON users.id = orders.user_id;
4. Self JOIN and Cross JOIN?
Self JOIN → Joins a table with itself.
Cross JOIN → Cartesian product of two tables.

Indexes & Performance

1. What is an Index?
An index improves query performance by speeding up data retrieval. 
CREATE INDEX idx_name ON users(name);
2. Types of Indexes?
Primary Index, Unique Index, Composite Index, Fulltext Index, Spatial Index. 
-- 1. PRIMARY INDEX: Unique identifier for table rows (PRIMARY KEY)
CREATE TABLE Users (
    UserID INT PRIMARY KEY,
    Name VARCHAR(50)
);

-- 2. UNIQUE INDEX: Ensures column values are unique
CREATE UNIQUE INDEX idx_email ON Users(Email);

-- 3. COMPOSITE INDEX: Index on multiple columns for faster search
CREATE INDEX idx_name_email ON Users(Name, Email);

-- 4. FULLTEXT INDEX: Optimized for text search in large text columns
CREATE FULLTEXT INDEX idx_bio ON Users(Bio);

-- 5. SPATIAL INDEX: Index for spatial data like coordinates or geometry
CREATE TABLE Locations (
    LocID INT PRIMARY KEY,
    Coordinates POINT,
    SPATIAL INDEX idx_coords (Coordinates)
);
3. What is EXPLAIN?
EXPLAIN is used to analyze query execution plans to optimize performance. Like Optimize the query executed time.
4. Difference between Normalization and Denormalization?
Normalization → Reduces redundancy, improves consistency.
Denormalization → Improves read performance by adding redundancy.
5. How to optimize a slow query?
✅ Use indexes
✅ Avoid SELECT *
✅ Use LIMIT
✅ Optimize JOINs and subqueries
✅ Use EXPLAIN to check query plan

Advanced / Tricky Questions

1. Difference between UNION and UNION ALL?
UNION → Removes duplicates.
UNION ALL → Keeps duplicates, faster. 
SELECT name FROM users
UNION
SELECT name FROM employees;
2. What is Subquery?
A query inside another query. 
SELECT * FROM users
WHERE id IN (SELECT user_id FROM orders);
3. Subquery vs JOIN?
Subquery → Query inside query, can be correlated.
JOIN → Combines tables, usually more efficient.
4. What is AUTO_INCREMENT?
Automatically generates unique numeric values for primary key columns.
5. Difference between stored procedures, functions, and triggers?

Stored Procedure → A set of SQL statements saved or stored in the database that you can reuse and run anytime. It may or may not return a value.

Function → A set of SQL statements saved in the database that you can reuse and run anytime, but it always returns a single value. Functions can be used in SQL statements like SELECT, WHERE.

Trigger → Executes automatically on table events (INSERT, UPDATE, DELETE).

-- 1. STORED PROCEDURE: Reusable set of statements
DELIMITER //
CREATE PROCEDURE IncreaseSalary(IN empID INT, IN amount DECIMAL(10,2))
BEGIN
    UPDATE Employees SET Salary = Salary + amount WHERE EmployeeID = empID;
END //
DELIMITER ;

-- Execute procedure
CALL IncreaseSalary(1, 500);

-- Can have multiple statements (UPDATE, INSERT, DELETE, etc.) and complex logic.
-------------------------------------------------

-- 2. FUNCTION: Returns a value, can be used in SELECT
DELIMITER //
CREATE FUNCTION GetSalary(empID INT)
RETURNS DECIMAL(10,2)
BEGIN
    DECLARE sal DECIMAL(10,2);
    SELECT Salary INTO sal FROM Employees WHERE EmployeeID = empID;
    RETURN sal;
END //
DELIMITER ;

-- Use function
SELECT Name, GetSalary(EmployeeID) AS Salary FROM Employees;

-- Use inside queries like subquery (SELECT Name, GetSalary(EmployeeID) AS Salary FROM Employees);
-------------------------------------------------

-- 3. TRIGGER: Executes automatically on table event
CREATE TRIGGER SalaryAudit
AFTER UPDATE ON Employees
FOR EACH ROW
BEGIN
    INSERT INTO SalaryLog(EmployeeID, OldSalary, NewSalary, ChangeDate)
    VALUES (OLD.EmployeeID, OLD.Salary, NEW.Salary, NOW());
END;
6. Explain the parameters in MySQL stored procedures?

IN → Input parameter = Passes a value into the procedure.

OUT → Output parameter = Procedure returns a value through it.

INOUT → Both input and output = Passes a value in, can modify it, and returns the new value.

7. Difference between Stored Procedure and Function in MySQL?

In MySQL:

  • Function → Must return a single value and cannot modify database tables. Can be used in SELECT, WHERE, etc.
  • Stored Procedure → May or may not return a value and can perform INSERT, UPDATE, DELETE or multiple operations.
8. What is a View?
A virtual table based on a query. Does not store data physically. 
CREATE VIEW IT_Employees AS
SELECT EmployeeID, Name, Salary
FROM Employees
WHERE Department = 'IT';

SELECT * FROM IT_Employees;
9. Difference between temporary and permanent tables?
Temporary → Exists only during session.
Permanent → Stored permanently in database. 
CREATE TEMPORARY TABLE temp_employees (
    EmployeeID INT,
    Name VARCHAR(50),
    Salary DECIMAL(10,2)
);
10. What is a composite key?

A composite key is a primary key that consists of two or more columns in a table, used together to uniquely identify each row.

Eg. PropertyID, CompanyID - The combination of PropID + CompanyID must be unique.
11. Difference between CHAR and VARCHAR?
CHAR → Fixed length.
VARCHAR → Variable length.

Security & Transactions

1. What is SQL Injection?
A security attack where malicious SQL code is injected into queries.
2. How to prevent SQL Injection?
Using prepared statements and parameterized queries. 
$stmt = $conn->prepare("SELECT * FROM users WHERE email = ?");
$stmt->execute([$email]);
3. What is a Transaction?
A transaction is a set of queries executed as a single unit. 
START TRANSACTION;
INSERT INTO orders VALUES (...);
COMMIT;
4. What are ACID Properties?
ACID ensures reliable database transactions: Atomicity, Consistency, Isolation, Durability. 
-- 1. ATOMICITY: "All or nothing" - either the entire transaction succeeds or fails
BEGIN TRANSACTION;
UPDATE Accounts SET Balance = Balance - 100 WHERE ID = 1;
UPDATE Accounts SET Balance = Balance + 100 WHERE ID = 2;
-- If any update fails, rollback so no money is transferred
ROLLBACK;

-- 2. CONSISTENCY: Database remains in a valid state after transaction
-- Example: Total balance before and after transfer remains the same
SELECT SUM(Balance) FROM Accounts;  -- Should be unchanged after valid transactions

-- 3. ISOLATION: Transactions do not interfere with each other
-- Example: Two users transferring money simultaneously won't cause incorrect balances

-- 4. DURABILITY: Once a transaction is committed, it is permanent
COMMIT;  -- Even if server crashes after this, changes remain

Quick summary of ACID:

✅ Atomicity: All or nothing / Rollback on failure

✅ Consistency: Data validity

✅ Isolation: Independent transactions or Multiple transactions do not conflict

✅ Durability: Permanent once committed or Once committed, changes remain even if server crashes
