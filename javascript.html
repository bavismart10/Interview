JavaScript Interview Questions

JavaScript Basics

1. What is JavaScript?
JavaScript is a client-side scripting language used to make web pages interactive and dynamic.
2. Difference between var, let, and const?
✅ var → Function-scoped, can be redeclared and reassigned.
✅ let → Block-scoped, cannot be redeclared but can be reassigned.
✅ const → Block-scoped, cannot be redeclared or reassigned.
3. What is Hoisting?
Hoisting moves variable and function declarations to the top of their scope before code execution.
4. What Temporal Dead Zone (TDZ)?
TDZ is the time period where a let or const variable exists but cannot be used before its declaration, and accessing it will throw a ReferenceError.
5. What are Data Types?
Primitive: String, Number, Boolean, Undefined, Null, Symbol, BigInt.
Non-Primitive: Object, Array, Function.
6. What are Prototype?

A prototype is the mechanism JavaScript uses to implement inheritance. Every JavaScript object can inherit properties and methods from another object through its prototype. (OR)

Prototype is the mechanism in JavaScript that allows objects to inherit properties and methods from other objects.

7. What is Event Propagation?

Event propagation is the mechanism that defines how events travel through the DOM, including capturing, target, and bubbling phases.

8. What are the phases of Event Propagation?

1. Capturing Phase -> Event moves from root to target element.

2. Target Phase -> Event reaches the target element.

3. Bubbling Phase -> Event bubbles from target back to root.

9. What is Event Bubbling?

Event bubbling is when an event triggered on a child element propagates upward to its parent elements.

10. What is Event Capturing?

Event capturing is when an event propagates from the root element down to the target element.

11. How do you stop event propagation?

Using event.stopPropagation().

12. Difference between event.stopPropagation() and event.preventDefault()?

stopPropagation() -> Stops event bubbling or capturing.

preventDefault() -> Prevents default browser behavior (eg: link navigation, form submit).

Functions & Scope

1. What is a Function?
A reusable block of code that performs a specific task.
2. Arrow Function vs Normal Function?
Arrow functions do not have their own this, normal functions do.
3. What is Closure?
A closure allows inner function to access variables from its outer function even after the outer function has executed.

Use Case : Counters, debounce() and throttle()

4. What is an Arrow Function?
Arrow functions are a shorter syntax for writing functions in ES6.

Example: Callbacks in array methods like map(), filter(), and reduce()

const add = (a, b) => a + b;
console.log(add(2,3)); // 5
5. What is a Callback Function?
A callback function is a function that is passed as an argument to another function and is executed inside that function.

Use Case: Handling asynchronous operations like API calls, event handling, or timers.

Example:

function greet(name, callback) {
  console.log("Hello, " + name);
  callback();
}

function afterGreeting() {
  console.log("Greeting has been displayed.");
}

greet("Bavi", afterGreeting);

// Output:
// Hello, Bavi
// Greeting has been displayed.
6. What is a Higher-Order Function (HOF)?

A Higher-Order Function is a function that:

✅ Takes one or more functions as arguments (or)

✅ Returns a function

Use Cases: Array methods -> .map(), .filter(), and .reduce(), setTimeout(), setInterval()

Example:

function greet(message) {
  return function(name) {
    console.log(`${message}, ${name}!`);
  };
}

const sayHello = greet("Hello");
sayHello("Bavi"); // Hello, Bavi!
7. What is IIFE (Immediately Invoked Function Expression)?

An IIFE is a function that runs immediately after it is defined.

IIFE makes variables private, so they don't conflict with global variables.

Use Case: Avoid Global Variables in Big Projects or Module pattern, counters, data privacy

Example:

var name = "Global";

(function() {
    var name = "Inside IIFE";
    console.log(name); // Inside IIFE
})();

console.log(name); // Global
8. What is Currying in JavaScript?

Currying is a technique of transforming a function with multiple arguments into a sequence of functions, each taking a single argument.

Currying transforms f(a, b, c) into f(a)(b)(c)

Use Case: function reusability, Partial function application, reusable and composable functions.

Example:

// Normal

function add(a, b, c) {
    return a + b + c;
}

console.log(add(2, 3, 4)); // 9

// Currying

// Generic multiply function
function multiply(a) {
    return function(b) {
        return a * b;
    };
}

var double = multiply(2); // 'a' is 2, returns function(b) waiting for 'b'
console.log(double(5)); // 2 * 5 = 10
console.log(double(10)); // 2 * 10 = 20

var triple = multiply(3); // 'a' is 3, returns function(b) waiting for 'b'
console.log(triple(5)); // 3 * 5 = 15

Asynchronous JavaScript

1. What is Asynchronous JavaScript?
It allows tasks like API calls to run in the background without blocking the main thread.
2. What is a Callback?
A function passed as an argument and executed later.
3. What is a Promise?
An object representing completion or failure of an async operation.
States: Pending, Fulfilled, Rejected.
4. Promise types

Promise.all → Waits for all promises to finish, fails if any reject. (Need all to resolve)

Promise.allSettled → Waits for all promises to finish, regardless of resolve/reject. (Need all to finish)

Promise.race → Waits for the first promise to finish, whether it resolves or rejects. (Need first to finish)

Promise.any → Waits for the first promise to successfully resolve. Fails only if all promises reject. (Need any to resolve)

5. What are async and await?
Keywords used to handle promises in a cleaner synchronous-looking way.

Arrays & Objects

1. Arrays vs Objects

Arrays → Ordered collection of items, accessed by numeric index.

Objects → Collection of key-value pairs, accessed by keys.

2. Common Array Methods?

map() → Creates a new array by applying a function to each element.

const numbers = [1,2,3];
const doubled = numbers.map(n => n * 2);
console.log(doubled); // [2,4,6]

filter() → Creates a new array with elements that pass a test.

const numbers = [1,2,3,4,5];
const even = numbers.filter(n => n % 2 === 0);
console.log(even); // [2,4]

reduce() → Reduces array to a single value by applying a function cumulatively.

const numbers = [1,2,3,4];
const sum = numbers.reduce((acc, curr) => acc + curr, 0);
console.log(sum); // 10

forEach() → Executes a function on each element (returns undefined).

const numbers = [1,2,3];
numbers.forEach(n => console.log(n * 2));
// Output: 2 4 6

push() → Adds elements to the end of an array.

const arr = [1,2];
arr.push(3);
console.log(arr); // [1,2,3]

pop() → Removes the last element from an array.

const arr = [1,2,3];
const last = arr.pop();
console.log(last); // 3
console.log(arr);  // [1,2]

shift() → Removes the first element from an array.

const arr = [1,2,3];
const first = arr.shift();
console.log(first); // 1
console.log(arr);   // [2,3]

unshift() → Adds elements to the beginning of an array.

const arr = [2,3];
arr.unshift(1);
console.log(arr); // [1,2,3]

slice() → Returns a shallow copy of a portion of an array.

const arr = [1,2,3,4];
const part = arr.slice(1,3);
console.log(part); // [2,3]

splice() → Adds/removes elements at a specific index.

const arr = [1,2,3,4];
arr.splice(1,2,9,10);
console.log(arr); // [1,9,10,4]

find() → Returns the first element that satisfies a test function.

const arr = [1,2,3,4];
const firstEven = arr.find(n => n % 2 === 0);
console.log(firstEven); // 2

findIndex() → Returns the index of the first element that satisfies a test.

const arr = [1,2,3,4];
const index = arr.findIndex(n => n % 2 === 0);
console.log(index); // 1

includes() → Checks if an array contains a specific element.

const arr = [1,2,3];
console.log(arr.includes(2)); // true

some() → Returns true if at least one element passes a test.

const arr = [1,3,5];
const hasEven = arr.some(n => n % 2 === 0);
console.log(hasEven); // false

every() → Returns true if all elements pass a test.

const arr = [2,4,6];
const allEven = arr.every(n => n % 2 === 0);
console.log(allEven); // true

sort() → Sorts elements in place (alphabetical or compare function).

const arr = [3,1,4,2];
arr.sort((a,b) => a-b);
console.log(arr); // [1,2,3,4]

reverse() → Reverses the elements of an array.

const arr = [1,2,3];
arr.reverse();
console.log(arr); // [3,2,1]

16. How Convert Object to Array and Back?

18. Shallow vs Deep Copy of Objects

Shallow Copy → When you clone an object, only the first-level properties are copied fully; nested objects/arrays still refer to the original object.

Changes in nested objects will affect the original, but top-level properties are independent.

const original = { 
  name: "Bavi", 
  address: { city: "Chennai", zip: 600001 } 
};

// Shallow copy using spread operator
const shallowCopy = { ...original };

// Changing top-level property
shallowCopy.name = "Ravi";
console.log(original.name); // "Bavi" → original unaffected

// Changing nested property
shallowCopy.address.city = "Bangalore";
console.log(original.address.city); // "Bangalore" → original affected!

Deep Copy → When you clone an object, all levels of properties are copied fully, creating a completely independent object.

Changes in nested objects will not affect the original object.

// Using JSON (simple method, ignores functions and undefined)
const deepCopy = JSON.parse(JSON.stringify(original));

deepCopy.address.city = "Mumbai";
console.log(original.address.city); // "Bangalore" → original unaffected

Object.values() → Returns array of an object's values.

Object.defineProperty(obj, 'a', { value: 10, writable: true });

const arr = [['a',1], ['b',2]];

Advanced Questions

1. What is the Call Stack?

The call stack in JavaScript is a data structure used by the JavaScript engine to manage and track the execution of functions in a program.

It follows LIFO (Last-In, First-Out) principle, meaning the last function added to the stack is the first one to be removed and executed

2. What is Event Delegation?

Event Delegation is a technique where you attach a single event listener to a parent element to handle events for its child elements, using event bubbling.

ul > li

  }

3. What is Event Loop?

5. What is Debouncing and Throttling?

Debouncing → Executes after the user stops the action for specified time.

Use Case: Search bar input/autocomplete. API call happens only after the user stops typing for 500ms.
function debounce(fn, delay) {
  let timer;

  return function () {
    clearTimeout(timer);
    timer = setTimeout(fn, delay);
  };
}

Throttling → Executes at regular intervals while action continues.

Use Case: Scroll events, Mouse move events, Multiple Button clicks. 
 
function throttle(fn, limit) {
  let last = 0;

  return function () {
    let now = Date.now();

    if (now - last >= limit) {
      last = now;
      fn();
    }
  };
}
6. What is Memory Leak?

Memory leak is when memory is not freed after it is no longer needed.

Common causes: timers like (setInterval) or events not cleared properly.

  console.log("Running...");

9. Difference between call, apply and bind?

call() → Invokes / execute the function immediately. Arguments are passed comma-separated: func.call(context, arg1, arg2).

apply() → Invokes / execute the function immediately. Arguments are passed as an array: func.apply(context, [arg1, arg2])

bind() → It is a method that creates a new function with a specific this value: const newFunc = func.bind(context, arg1). Useful for events or timers.

One Line

call() → Invokes immediately, arguments separately.
apply() → Invokes immediately, arguments as array.
bind() →Returns new function.
