PHPUnit Interview Questions

1. What is PHPUnit?

PHPUnit is a testing framework for PHP used to write and run unit tests to verify that application code works as expected.

One Line : Framework for unit testing in PHP. (vendor/bin/phpunit)

PHPUnit is used in real projects to test application logic automatically before deployment.

Example : In my projects, I use PHPUnit to test backend logic like API validation, database operations, and business rules. It helps catch bugs early and ensures stability before deployment, especially when integrated with CI/CD pipelines

To install PHPUnit, I use Composer: composer require --dev phpunit/phpunit. Then I create a test class extending TestCase and write test methods using assertions like assertEquals() to validate expected output
2. Why is PHPUnit used?

To find bugs early, improve code quality, and ensure application stability.

3. What is Unit Testing?

Testing individual units or functions of an application separately.

4. What is Test Case in PHPUnit?

A test case is a class that contains test methods used to verify application functionality.

5. How to install PHPUnit?
composer require --dev phpunit/phpunit
6. Basic PHPUnit Test Example?
use PHPUnit\Framework\TestCase;

class SampleTest extends TestCase {
  public function testSum() {
    $this->assertEquals(4, 2+2);
  }
}
7. What is assert in PHPUnit?

Assert methods are used to verify expected and actual results.

Examples: assertEquals(), assertTrue(), assertFalse(), assertNull()

8. Common assert methods?

assertEquals(), assertTrue(), assertFalse(), assertNull(), assertNotNull(), assertCount()

9. What is setUp() method?

The setUp() method is a special method in PHPUnit that runs before every test method to initialize test data. It is used to clean up resources created during the test.

Example: Close database connections, Delete temporary files, Reset variables.

10. What is tearDown() method?

tearDown() is a special method that runs after every test method..

11. What is Mocking?

Mocking is a testing technique where we create a fake object to simulate the behavior of a real object.

One Line : Creating fake objects to isolate tests.

12. Why use Mock objects?

To isolate unit tests and avoid database, API, and external service dependencies. Eg. Sends email

13. What is Code Coverage?

It measures how much of application code is tested by your test cases.

One Line : It shows the percentage of code executed when running tests.

14. How to generate code coverage report?
phpunit --coverage-html coverage/
15. Difference between Unit Testing and Integration Testing?

Unit Testing -> Tests single function or class.

Integration Testing -> Tests combined modules.

16. What is Data Provider?

Used to pass multiple test data sets to a single test method.

It allows you to run the same test multiple times with different input values.

17. Data Provider Example?
/**
* @dataProvider sumProvider
*/
public function testSum($a, $b, $expected) {
  $this->assertEquals($expected, $a+$b);
}

public function sumProvider() {
  return [
    [1,2,3],
    [2,3,5]
  ];
}
18. What is PHPUnit configuration file?

phpunit.xml is used to configure test execution and settings.

<?xml version="1.0" encoding="UTF-8"?>
<phpunit bootstrap="vendor/autoload.php">
    <testsuites>
        <testsuite name="Application Test Suite">
            <directory>tests</directory>
        </testsuite>
    </testsuites>
</phpunit>
19. How to run PHPUnit tests?
vendor/bin/phpunit
20. Real-time usage of PHPUnit?

Testing APIs, validating business logic, regression testing, CI/CD automation.
