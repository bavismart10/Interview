QA & Automation - Playwright Interview Questions

Fresher Level Questions

1. What is Playwright?
Playwright is an open-source end-to-end automation framework developed by Microsoft for testing modern web applications across Chromium, Firefox, and WebKit browsers.
2. Which languages does Playwright support?
JavaScript, TypeScript, Python, Java, and C#.
3. What browsers are supported?
Chromium, Firefox, and WebKit (Safari engine).
4. What is Locator in Playwright?
Locator is the core way to find elements with built-in auto-waiting and retry. 
page.locator('#id')
page.getByText('Submit')
5. What is Auto-Waiting?
Playwright automatically waits for elements to be visible, stable, and enabled before performing actions.
6. Basic Actions?
locator.click()          → Clicks on the element.
locator.fill()           → Fills an input field with text.
locator.check()          → Checks a checkbox or radio button.
locator.uncheck()        → Unchecks a checkbox.
locator.hover()          → Moves the mouse over the element.
locator.selectOption()   → Selects an option from a dropdown.
locator.setInputFiles()  → Uploads files to a file input field.
7. What are Assertions?
Assertions are used to validate expected results in a test.

Assertions in Playwright are built-in checks (using expect) that verify if the application behaves as expected during a test. 

await expect(page).toHaveTitle('Home');  
// Verifies that the page title is "Home".

await expect(locator).toBeVisible();  
// Checks that the element is visible on the page.

await expect(locator).toHaveText('Success');  
// Confirms the element contains the text "Success".

await expect(locator).toHaveValue('John');  
// Ensures the input field has the value "John".

await expect(page).toHaveURL('https://example.com');  
// Verifies that the current page URL matches the given URL.

Pick Locator → Used when you need to target a specific element (button, input, link, checkbox, etc.) before performing actions or validations.

const loginButton = page.locator('#login');
await loginButton.click();

Assert Visibility → Verifies that an element is visible to the user.

await expect(page.locator('.success-message')).toBeVisible();

Assert Text → Checks whether an element contains the expected text.

await expect(page.locator('h1')).toHaveText('Dashboard');

Assert Value → Confirms that an input field holds the expected value.

await expect(page.locator('#email')).toHaveValue('test@example.com');

Assert Snapshot → Compares the current UI with a previously saved snapshot to detect changes.

await expect(page).toHaveScreenshot();
8. What is Browser, Context, and Page?
Browser - Browser instance. BrowserContext - Isolated session (like incognito). Page - Single tab inside context.
9. Important Playwright Commands?
npx playwright test          → Runs all Playwright test cases.
npx playwright test --ui     → Run tests using Playwright UI mode
npx playwright test --debug  → Run tests in debug mode (opens Inspector)
npx playwright codegen       → Opens Playwright Codegen to record test scripts.
npx playwright show-report   → Opens the last generated HTML test report.
npx playwright test --headed → Run tests in headed (visible browser) mode

npx playwright test --project=chromium → Run tests in a specific browser
npx playwright test --retries=2        → Retry failed tests
npx playwright show-trace trace.zip    → Open Playwright Trace Viewer
npx playwright help                    → Show all available commands

Intermediate Level Questions

1. What is Playwright Test?
Built-in test runner supporting fixtures, hooks, parallel execution, retries, and reporting.
2. Explain Hooks in Playwright.
test.beforeAll()    → Runs once before all tests in the file.
test.beforeEach()   → Runs before each individual test.
test.afterEach()    → Runs after each individual test.
test.afterAll()     → Runs once after all tests in the file.
Used for setup and teardown.
3. How do you handle multiple tabs?
const newPage = await context.waitForEvent('page');
4. How do you handle iframes?
await page.frameLocator('#frameId').locator('button').click();
5. How do you intercept or mock API calls?
await page.route('**/api/**', route =>
  route.fulfill({
    status: 200,
    body: JSON.stringify({ success: true })
  })
);
6. How do you handle authentication?
Use storageState: 
await context.storageState({ path: 'state.json' });
Reuse login session in tests.
7. How do you run tests in parallel?
Configure workers in playwright.config.js: 
workers: 4
8. What is Trace Viewer?
Helps debug failures with screenshots and network logs. 
npx playwright show-trace trace.zip

Advanced Level Questions

1. Explain Page Object Model (POM).
Design pattern where page elements and methods are stored in separate classes to improve reusability and maintainability.
2. How do you perform API testing in Playwright?
Using APIRequestContext: 
const response = await request.get('/api/users');
expect(response.status()).toBe(200);
3. How do you perform visual testing?
await expect(page).toHaveScreenshot();
await expect(locator).toMatchSnapshot();
4. How do you handle Shadow DOM?
Playwright supports shadow DOM automatically with locators. 
page.locator('css=custom-element >> button')
5. How do you handle network offline or throttling?
await context.setOffline(true);
6. How do you structure a real-time Playwright framework?
  • Page Object Model
  • Reusable utilities
  • Environment-based config
  • CI/CD integration
  • HTML reporting
  • Parallel execution
