PHPUnit & Agile Interview Questions

PHPUnit (Unit Testing)

1. What is PHPUnit?
PHPUnit is a unit testing framework for PHP used to test individual units like methods and classes.
2. Why is PHPUnit important?
• Catches bugs early
• Improves code quality
• Makes refactoring safe
• Essential for CI/CD
• Shows maintainable code
3. What is Unit Testing?
Testing small units of code (functions, methods) independently.
4. Basic PHPUnit Test Example
use PHPUnit\Framework\TestCase;

class MathTest extends TestCase {

    public function testAddition() {
        $this->assertEquals(4, 2 + 2);
    }
}
5. Common PHPUnit Assertions
assertEquals()
assertTrue()
assertFalse()
assertNull()
assertCount()
assertInstanceOf()
expectException()
6. How to run PHPUnit?
Run command in terminal: 
vendor/bin/phpunit tests

Agile Methodology

7. What is Agile?
Agile is a software development methodology based on iterative and incremental development.
8. Core Agile Concepts
• Iterative Development
• Incremental Delivery
• Continuous Feedback
• Adaptability to Change
9. What is Scrum?
Scrum is a popular Agile framework that manages work in short cycles called Sprints.
10. What is a Sprint?
A Sprint is a short time period (1–2 weeks) where a small set of work is completed.
11. Important Scrum Meetings
• Sprint Planning
• Daily Stand-up
• Sprint Review
• Sprint Retrospective
• Backlog Refinement
12. What is Product Backlog?
A prioritized list of tasks/features to be completed in the project.
13. What is Retrospective?
A meeting where the team discusses what went well and how to improve.

Agile + PHPUnit + CI/CD

14. How does PHPUnit support Agile?
Automated tests ensure quick feedback in every sprint.
15. What is CI/CD?
CI → Continuous Integration
CD → Continuous Delivery/Deployment
16. Why testing is important in CI/CD?
Automated tests run before deployment to prevent production bugs.
