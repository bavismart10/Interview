jQuery & AJAX Interview Questions

jQuery Basics

1. What is jQuery?
jQuery is a fast, lightweight JavaScript library that simplifies DOM manipulation, event handling, animations, and AJAX.
2. Why use jQuery?
It reduces JavaScript code, handles cross-browser issues, and makes AJAX calls easier.
3. Difference between JavaScript and jQuery?
JavaScript is a programming language, while jQuery is a library built on JavaScript to simplify tasks.
4. What is $(document).ready()?
It ensures the DOM is fully loaded before executing jQuery code.
5. How do you select elements in jQuery?
Using CSS selectors like $('#id'), $('.class'), and $('tag').
6. Difference between id and class selector?
id selects a unique element, class selects multiple elements.
7. What is chaining in jQuery?
Chaining allows multiple jQuery methods to be called on the same element in one line.
$("#box").css("color", "red").slideUp(500).slideDown(500);
8. Common jQuery methods?
hide(), show(), toggle(), addClass(), removeClass(), css()

AJAX Interview Questions

9. What is AJAX?
AJAX allows sending and receiving data from the server without reloading the web page.
10. Full form of AJAX?
Asynchronous JavaScript and XML.
11. How AJAX works?
AJAX sends a request to the server in the background and updates the page using the response.
12. Difference between synchronous and asynchronous?
Synchronous waits for a response, asynchronous does not block page execution.
13. What is $.ajax()?
$.ajax() is a jQuery method used to make AJAX requests with full control.
14. Difference between $.get() and $.post()?
$.get() sends data via URL, $.post() sends data in request body.
15. What data formats are used in AJAX?
JSON, XML, HTML, and plain text.
16. What is success and error callback?
success runs when the request is successful, error runs when the request fails.

AJAX Code Example

$.ajax({
  url: "fetch.php",
  type: "POST",
  data: { id: 1 },
  success: function(response) {
    console.log(response);
  },
  error: function() {
    alert("Error occurred");
  }
});

Advanced / Tricky Questions

17. What is CORS in AJAX?
CORS (Cross-Origin Resource Sharing) is a browser security feature that controls how resources are shared between different origins (domains).
18. What is event delegation?
A parent element event listener handles events for its child elements.
19. What is dataType in AJAX?
It specifies the type of data expected from the server, like JSON or HTML.
20. Difference between $.ajax() and fetch()?
$.ajax() is jQuery-based, while fetch() is native JavaScript and promise-based.
21. What is Event Listener?
An event listener is a function that waits for a specific event and executes code when that event occurs. Example: addEventListener()
21. Difference between .prop() and .attr()?

.attr() -> Used to get the HTML attribute value. Eg.id, class, src, href,data-.

.prop() -> Used to get the current DOM property value.. Eg.checked,selected,disabled,readonly.

22. What is $.noConflict()?

Used to avoid conflicts when multiple JavaScript libraries use the $ symbol.

23. Difference between .empty(), .remove() and .detach()?

.empty() -> Removes only child elements.

.remove() -> Removes element and its data/events.

.detach() -> Removes element but keeps its data/events.

24. Difference between .stop(), .delay() and .queue()?

.stop() -> Stops currently running animation.

.delay() -> Delays animation execution.

.queue() -> Manages animation queue.

25. What are HTTP status codes?

200 -> OK

400 -> Bad Request

401 -> Unauthorized

403 -> Forbidden

404 -> Not Found

500 -> Server Error

26. Difference between synchronous and asynchronous AJAX?

Synchronous -> Blocks UI until response arrives.

Asynchronous -> Runs in background without blocking UI.
