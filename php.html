24. What are the rules for naming a PHP variable?
Start with a dollar sign: All PHP variables must begin with the $ symbol.
Initial character: The first character after the $ sign must be a letter (A-z) or an underscore (_). It cannot be a number.
Allowed characters: The rest of the variable name can only contain alphanumeric characters and underscores (A-z, 0-9, and _).
No spaces: Variable names cannot contain spaces.
Case sensitivity: PHP variable names are case-sensitive. For example, $age and $AGE are treated as two different variables.
Reserved words: Do not use PHP reserved keywords (like function, class, echo, print) as variable names to avoid errors.
Special variables: $this is a special, predefined variable used within classes and cannot be assigned a value or referenced dynamically in the same way as user-defined variables.

4. Difference between interface and abstract class?

In Short:

Interface → Only method declarations but cannot provide method implementations; It supports multiple inheritance.

Abstract class → Can have both method declaration and implementation; single inheritance.

Interface → Interfaces define methods that must be implemented by a class, but cannot provide method implementations.

eg. Payment Gateway. Which is implement by paypal, gateway methods

interface PaymentGateway {
    public function pay($amount); // Method Declaration
}

class Paypal implements PaymentGateway {
    public function pay($amount) {
        return "Paid $amount using PayPal";
    }
}

class Stripe implements PaymentGateway {
    public function pay($amount) {
        return "Paid $amount using Stripe";
    }
}

Abstract Class → Abstract classes can define methods that must be implemented (abstract methods) and also provide method implementations (concrete methods).

Abstract class: Defines abstract methods (child must implement) and partial concrete methods (already implemented methods child can use)

eg. Payment Gateway rules with base payment logic 

abstract class Payment {
    abstract public function pay($amount); // Method Declaration
    public function log($msg) { // Method Implementation
        return "Log: $msg";
    }   
}

class Paypal extends Payment {
    public function pay($amount) {
        return "Paid $amount using PayPal";
    }
}

One Line :

✅ interface: only provides method declarations, does not provide method implementations.

✅ Abstract class: provides both method declarations (abstract methods) and method implementations (concrete methods)

13. What is Method Overriding?

Method Overriding →It is mechanism / technique, when both parent and child classes have the same method name, the child overrides the parent's method behavior.

Example : Used when we want to perform similar operations with the same method name but with different parameters.

class ParentClass {
    public function greet() {
        echo "Hello from Parent";
    }
}

class ChildClass extends ParentClass {
    public function greet() {
        echo "Hello from Child";
    }
}

$obj = new ChildClass();
$obj->greet(); // Output: Hello from Child

What is Operator Precedence in PHP?

Operator Precedence → It decides which operator executes first in an expression.

Example:

$result = 5 + 3 * 2;
echo $result; // 11

Explanation → Multiplication (*) runs first, then addition (+).

You can use brackets to change the order:

$result = (5 + 3) * 2;
echo $result; // 16

Brackets () have higher priority.

What are References (&) in PHP?

Reference (&) → A reference allows two variables to point to the same memory location, so changing one variable also changes the other.

References allow two variables to share the same memory, making changes to one affect the other.

Example:

$a = 10;
$b = &$a;

$b = 20;
echo $a; // 20

Here, $b is a reference to $a, so changing one changes the other.

Without & → It copies the value.

With & → It shares the same memory.

Important PHP String Functions

Common String Functions/Methods in PHP

1. strlen() → Returns the length of a string.

$str = "Hello";
echo strlen($str); // 5

2. str_replace() → Replaces all occurrences of a search string with a replacement.

echo str_replace("World", "PHP", "Hello World"); // Hello PHP

3. strpos() → Returns the position of the first occurrence of a substring.

echo strpos("Hello World", "World"); // 6

4. substr() → Returns a part of the string.

echo substr("Hello World", 0, 5); // Hello

5. strtolower() → Converts string to lowercase.

echo strtolower("HELLO"); // hello

6. strtoupper() → Converts string to uppercase.

echo strtoupper("hello"); // HELLO

7. trim() → Removes whitespace (or other characters) from both ends of a string.

$str = "  Hello  ";

8. explode() → Splits string into an array by a delimiter.

$str = "apple,banana,orange";
print_r(explode(",", $str));

9. implode() → Joins array elements into a string.

$arr = ["apple","banana"];
echo implode("-", $arr); // apple-banana

10. str_repeat() → Repeats a string multiple times.

echo str_repeat("Hi", 3); // HiHiHi

11. str_pad() → Pads a string to a certain length.

echo str_pad("Hello", 10, "*"); // Hello*****

14. join() / implode() → Joins array elements into a string.

15. lcfirst() → Converts first character of string to lowercase.

18. ltrim() → Removes whitespace (or other characters) from the left side of string.

19. rtrim() → Removes whitespace (or other characters) from the right side of string.

20. str_split() → Splits a string into an array of individual characters (or fixed-length chunks).

21. strrev() → Reverses a string.

$str = "Hello";
echo strrev($str); // olleH

SOLID principles

Explain the SOLID principles in PHP?
The SOLID principles are five design rules for writing better, maintainable object-oriented code.

1. Single Responsibility Principle (SRP):

Memory Tip: One class, one job

A class should have only one job. (One responsibility only inside Class)

Example: I separated database, business logic, and notifications → SRP

class Logger {
    public function log($message) {
        echo $message;
    }
}
// It is doing only logging.
2. Open/Closed Principle (OCP):

Memory Tip: Open for extension, closed for modification

You can add new features or features or functionality without changing the existing code.

Example: Used strategy pattern for salary calculation → OCP

interface Shape {
    public function area();
}

class Rectangle implements Shape {
    public function area() { return 20; }
}

class Circle implements Shape {
    public function area() { return 30; }
}
// We can add new shapes without changing existing code
3. Liskov Substitution Principle (LSP):

Memory Tip: Child can replace parent

Child classes should work without breaking the parent class behavior.

LSP: You should be able to replace or substitute a parent class with a child class, and the program should still work correctly.

Example: Ensured all employee types follow same contract

✅ Followed LSP: You swap the parent with the child → program still works.

❌ Violated LSP: You swap the parent with the child → program behaves incorrectly, crashes, or gives

class Bird { 
    public function fly() {} 
}
class Ostrich extends Bird {
    public function fly() {
        throw new Exception("Ostrich can't fly!");
    }
}
// Violation: Ostrich can't fly like Bird. LSP says avoid this.
4. Interface Segregation Principle (ISP):

Memory Tip: Many small interfaces, not one big

Don't force a class to implement methods it doesn't use. Means If you have an interface with 5 methods, but your class only needs 2 of them, don't force the class to implement all 5.

Example: Split large interfaces into smaller ones

interface Worker {
    public function work();
}

interface Sleeper {
    public function sleep();
}

class HumanWorker implements Worker, Sleeper {
    public function work() { echo "Working"; }
    public function sleep() { echo "Sleeping"; }
}

class RobotWorker implements Worker {
    public function work() { echo "Working"; }
}

// Robots don't need sleep method. So we separated the interface.
5. Dependency Inversion Principle (DIP):

Memory Tip: Depend on abstractions, not concretions

High-level modules (like App) should not depend on low-level modules (like MySQLDatabase) to work. Instead, make both depend on an abstraction (an interface).

Example: The app doesn't care which database it uses, it just cares that it can connect.

interface Database {
    public function connect();
}

class MySQLDatabase implements Database {
    public function connect() { echo "Connected to MySQL"; }
}

class App {
    private $db;
    public function __construct(Database $db) {
        $this->db = $db;
    }
    public function run() { $this->db->connect(); }
}

$app = new App(new MySQLDatabase());
$app->run();
