PHP Interview Questions

What is PHP?

PHP is an open-source, free, and powerful server-side scripting language used to create dynamic and interactive web pages.

Security

1. What is SQL Injection?
SQL Injection is a security vulnerability where an attacker injects malicious SQL code into database queries.
2. How to prevent SQL Injection?
Use Prepared Statements (PDO/MySQLi), validate inputs, avoid direct query insertion, and limit DB privileges.
3. What is XSS?
Cross-Site Scripting (XSS) is an attack where malicious scripts are injected into web pages.
4. How to prevent XSS?
Use htmlspecialchars(), validate inputs, enable Content Security Policy (CSP), and escape output properly.

OOPS in PHP

5. What is OOPS?
OOPS (Object-Oriented Programming System) is a programming approach where we write code using classes and objects.
6. What is Encapsulation?
Encapsulation means wrapping data and methods inside a class and restricting direct access using access modifiers.
7. What is Inheritance?
Inheritance allows one class to acquire properties and methods of another class using extends. Example: Base Model, Base Controller.
8. What is Polymorphism?
Polymorphism means same method name but different behavior depending on the object.
9. What is Abstraction?
Abstraction hides implementation details and shows only essential functionality. Example: Model called inside Controller.
Pillar Meaning
Encapsulation Data hiding using access modifiers
Inheritance Parent → Child relationship
Polymorphism Same method, different behavior
Abstraction Hide implementation details
10. What are Access Modifiers?
public → Accessible anywhere
protected → Class + Child class
private → Same class only
11. What is Constructor?
Constructor (__construct) is automatically called when an object is created.
12. What is Destructor?
Destructor (__destruct) is called when object is destroyed.
13. What is static?
Static belongs to class, not object. Access using ::
14. What is Trait?
Trait allows code reuse across multiple classes.
15. Difference between self and $this?
$this → Refers to current object.
self:: → Refers to current class.

PHP Design Patterns

16. What are Design Patterns?
Design patterns are reusable solutions to common software design problems that improve maintainability, scalability, and testability.

Creational Patterns

Singleton → Ensures only one instance (Example: DB connection)
Factory → Creates objects without exposing logic (Example: Payment gateway)
Builder → Builds complex objects step by step

Structural Patterns

MVC → Separates Model, View, Controller
Adapter → Makes incompatible interfaces work together
Decorator → Adds behavior without modifying class
Facade → Simple interface to complex system

Behavioral Patterns

Observer → Event listeners / Notifications
Strategy → Change algorithm dynamically
Command → Encapsulates actions as objects

Magic Methods

__construct() Automatically called when an object is created. Used to initialize properties.
__destruct() Automatically called when object is destroyed. Used to close connections or cleanup.
__get() Called when accessing inaccessible/private property. Handles undefined property read.
__set() Called when setting inaccessible/private property. Handles undefined property write.
__toString() Called when object is treated as a string. Returns string representation of object.

PHP 7 vs PHP 8 Differences (With Explanation & Examples)

Feature PHP 7 PHP 8
Saner String to Number Comparisons Non-numeric strings were converted to 0 during loose comparison. 
var_dump(0 == "abc"); // true
Non-numeric strings are NOT converted to 0. Safer comparison behavior. 
var_dump(0 == "abc"); // false
Constructor Property Promotion More boilerplate code (property declared separately). 
class User {
    public $name;

    public function __construct($name) {
        $this->name = $name;
    }
}
Less boilerplate code (declare & assign in constructor). 
class User {
    public function __construct(
        public string $name
    ) {}
}
Type Declarations Scalar type declarations introduced (int, float, string, bool). 
function sum(int $a, int $b) {
    return $a + $b;
}
Union types allow multiple data types. 
function test(int|string $data) {
    return $data;
}
Null Handling Null coalescing operator (??) to provide default values. 
$name = $_GET['name'] ?? 'Guest';
Nullsafe operator (?->) to avoid null errors. 
$user?->profile?->name;
Named Arguments Only positional arguments supported. 
function user($name, $age) {}
user("Bavi", 25);
Named arguments introduced. 
function user($name, $age) {}
user(age:25, name:"Bavi");
Switch Alternative Only switch statement available. 
switch($x) {
    case 1:
        echo "One";
        break;
}
Match expression introduced (strict comparison).It performs strict comparison and does not require a break statement. 
echo match($x) {
    1 => "One",
    default => "Other",
};
Annotations Doc comments used for metadata. 
/**
 * @Route("/home")
 */
Attributes (#[...]) introduced. 
#[Route('/home')]

PHP 8 improves error handling by converting many warnings into exceptions like TypeError and ValueError, making debugging easier and allowing errors to be handled using try-catch blocks compared to PHP 7.

Important PHP Questions (3 Years Experience)

17. Difference between include and require?
include → Warning if file missing.
require → Fatal error if file missing.
18. What is Namespace?
Namespace avoids class name conflicts in large applications.
19. What is Composer?
Composer is a dependency manager for PHP.
20. What is Autoloading?
Automatically loads class files using spl_autoload_register() or Composer PSR-4.
21. What is Session and Cookie?
Session → Stored on server.
Cookie → Stored in browser.
22. What is REST API in PHP?
REST API allows client-server communication using HTTP methods.
23. What is PDO?
PDO (PHP Data Objects) provides secure database access using prepared statements.
24. What is Dependency Injection?
Dependency Injection means passing required objects to a class instead of creating them inside.
25. How to improve PHP performance?
Use caching, optimize queries, use indexes, enable OPcache, reduce DB calls, and avoid unnecessary loops.

Microservices

26. What is Microservices?
Microservices is an architecture where an application is divided into small independent services that communicate via APIs.
