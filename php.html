What is Operator Precedence in PHP?

Operator Precedence ‚Üí It decides which operator executes first in an expression.

Example:

$result = 5 + 3 * 2;
echo $result; // 11

Explanation ‚Üí Multiplication (*) runs first, then addition (+).

You can use brackets to change the order:

$result = (5 + 3) * 2;
echo $result; // 16

Brackets () have higher priority.

What are References (&) in PHP?

Reference (&) ‚Üí It allows two variables to point to the same value in memory.

Example:

$a = 10;
$b = &$a;

$b = 20;
echo $a; // 20

Here, $b is a reference to $a, so changing one changes the other.

Without & ‚Üí It copies the value.

With & ‚Üí It shares the same memory.

SOLID principles

Explain the SOLID principles in PHP?
The SOLID principles are five design rules for writing better, maintainable object-oriented code.

1. Single Responsibility Principle (SRP):

Memory Tip: One class, one job

A class should have only one job. (One Method only inside Class) Example: 
class Logger {
    public function log($message) {
        echo $message;
    }
}
// It is doing only logging.
2. Open/Closed Principle (OCP):

Memory Tip: Open for extension, closed for modification

You can add new features or featuresfunctionality without changing existing code. Example: 
interface Shape {
    public function area();
}

class Rectangle implements Shape {
    public function area() { return 20; }
}

class Circle implements Shape {
    public function area() { return 30; }
}
// We can add new shapes without changing existing code
3. Liskov Substitution Principle (LSP):

Memory Tip: Child can replace parent

Child classes should work without breaking the parent class behavior. Example: 
class Bird { 
    public function fly() {} 
}
class Ostrich extends Bird {
    public function fly() {
        throw new Exception("Ostrich can't fly!");
    }
}
// Violation: Ostrich can't fly like Bird. LSP says avoid this.
4. Interface Segregation Principle (ISP):

Memory Tip: Many small interfaces, not one big

Don't force a class to implement methods it doesn't use. Means If you have an interface with 5 methods, but your class only needs 2 of them, don't force the class to implement all 5.

Example: 
interface Worker {
    public function work();
}

interface Sleeper {
    public function sleep();
}

class HumanWorker implements Worker, Sleeper {
    public function work() { echo "Working"; }
    public function sleep() { echo "Sleeping"; }
}

class RobotWorker implements Worker {
    public function work() { echo "Working"; }
}

// Robots don't need sleep method. So we separated the interface.
5. Dependency Inversion Principle (DIP):

Memory Tip: Depend on abstractions, not concretions

High-level modules (like App) should not depend on low-level modules (like MySQLDatabase) to work. Instead, make both depend on an abstraction (an interface).

Example: The app doesn't care which database it uses, it just cares that it can connect. 
interface Database {
    public function connect();
}

class MySQLDatabase implements Database {
    public function connect() { echo "Connected to MySQL"; }
}

class App {
    private $db;
    public function __construct(Database $db) {
        $this->db = $db;
    }
    public function run() { $this->db->connect(); }
}

$app = new App(new MySQLDatabase());
$app->run();
One liner 
S - One class, one job

O - Extend, don't modify

L - Child can replace parent

I - Don't force unused methods

D - Depend on interfaces, not concrete classes

        echo "Bark";
    }
}

Abstract Class ‚Üí It can have both method declarations and partial implementation method implementations.

abstract class Animal {
    abstract public function sound(); // Declaration

    public function eat() { // Implementation
        echo "Eating";
    }
}

class Dog extends Animal {
    public function sound() {
        echo "Bark";
    }
}

What are the rules for naming a PHP variable?
Start with a dollar sign: All PHP variables must begin with the $ symbol.
Initial character: The first character after the $ sign must be a letter (A-z) or an underscore (_). It cannot be a number.
Allowed characters: The rest of the variable name can only contain alphanumeric characters and underscores (A-z, 0-9, and _).
No spaces: Variable names cannot contain spaces.
Case sensitivity: PHP variable names are case-sensitive. For example, $age and $AGE are treated as two different variables.
Reserved words: Do not use PHP reserved keywords (like function, class, echo, print) as variable names to avoid errors.
Special variables: $this is a special, predefined variable used within classes and cannot be assigned a value or referenced dynamically in the same way as user-defined variables.
